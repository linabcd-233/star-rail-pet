# star-rail-pet — 架构与目录规划

本文档侧重 **Python 预览线**的目录与模块边界（拆分已完成）。**该线当前已搁置维护**（不接新功能）；**桌宠与迭代以 `pet-web/` 为准**。**桌宠产品交互**（番茄钟、穿透、目光等）在 **`pet-web/`**，阶段划分见 **`docs/前端模块拆分与交互落地规划.md`**；产品规则见 **`docs/交互与表情仲裁设计.md`**。  
（可行性、技术选型背景见 **`docs/技术文档.md`**。）

## 1. 当前仓库分析

### 1.1 文件一览（已按本规划拆分）

| 路径 | 职责 |
|------|------|

| `src/star_rail_pet/spine/draw.py` | `mesh_draws_for_bones`、`build_mesh_draw_list` |
| `src/star_rail_pet/render/gl_preview.py` | OpenGL + pygame 预览循环 |
| `src/star_rail_pet/anim/sample.py` | 动画线性采样 |
| `src/star_rail_pet/preview_entry.py` | CLI 入口 |
| `tools/preview_spine_character.py` | 薄封装入口（设置 `PYTHONPATH`） |
| 根目录 `preview_spine_character.py` | 兼容旧命令，转发至 `preview_entry` |
| 根目录 `spine_anim_sample.py` | 兼容旧 `import`，转发至包内 |
| `preview_skeleton.py` | DragonBones ske 线框预览 |
| `pyproject.toml` | 可 `pip install -e .` 安装包 |
| `docs/技术文档.md` | 可行性、选型背景、预览说明 |

### 1.2 素材位置

- 默认目录：**`assets/argenti/`**（已自网盘 `银技` 复制 json/atlas；`1302.png` 若缺失可用 pygame 生成占位图，**正式显示请替换为与 atlas 匹配的完整 PNG**）。
- CLI 默认 `--dir` 指向 `assets/argenti`，仍可用 `--dir` 覆盖为任意路径。

## 3. 目标目录结构（建议）

```
star-rail-pet/
├── pet-web/                     # Tauri + Vite + Spine 官方 WebGL（与 Python 并行）
│   ├── scripts/sync-assets.mjs  # 将 assets/argenti 同步到 public/argenti
│   ├── public/argenti/          # 同步后的资源（勿手改为主数据源）
│   ├── src-tauri/               # Rust 壳、窗口、打包
│   └── src/                     # 前端：main.ts 组装；Spine 应用在 spine/canvasApp.ts 等
├── assets/                      # 不入 Git 的大文件可用 .gitignore + README 说明放置方式
│   └── argenti/                 # 银枝 Spine 资源（示例名，可改）
│       ├── 1302.1a88ff13.json   # Spine 4.2 骨骼与动画
│       ├── 1302.atlas
│       ├── 1302.png
│       └── README.txt           # 可选：来源、版本、授权自用说明
│   # 可选：assets/audio/ / assets/_ske_backup/ 等
│
├── src/
│   └── star_rail_pet/           # 包名（可改为 pet / srp 等，需与 import 一致）
│       ├── __init__.py
│       ├── spine/               # Spine 运行时（精简自研）
│       │   ├── bones.py         # BoneWorld、compute_bone_worlds、继承/剪切等
│       │   ├── constraints.py  # transform 约束等
│       │   ├── atlas.py         # parse_atlas、AtlasRegion、UV 辅助
│       │   ├── mesh_scene.py    # load_spine_mesh_scene、resolve_attachment、MeshSlotStatic
│       │   └── draw.py          # mesh_draws_for_bones、build_mesh_draw_list（纯数据侧）
│       ├── anim/
│       │   └── sample.py        # 自 spine_anim_sample 迁入或 re-export
│       ├── render/              # 可选：再拆 OpenGL / pygame 窗口
│       │   └── gl_preview.py    # run_gl_preview、纹理与着色
│       └── preview/             # 后续：procedural 状态机、输入
│           └── __init__.py      # 占位
│
├── tools/                       # 仅开发/调试用入口，轻量
│   └── preview_spine_character.py   # 薄封装：argparse + 调 src 中 API
│
├── docs/                        # 专题文档（列表见根目录 README「文档总索引」）
│   ├── 架构规划.md              # 本文件
│   ├── 技术文档.md
│   └── …                        # 导读、交互设计、纪要、拆分规划等
├── preview_skeleton.py          # 根目录：DragonBones ske 线框（可选）
├── requirements.txt
├── environment.yml
└── README.md                    # 项目简介、命令、文档总索引
```

**说明**：

- **`assets/`**：你把网盘资源复制进 `assets/argenti/`（或等价路径）后，默认命令改为例如 `--dir assets/argenti`，便于协作与 CI（大文件可 Git LFS 或仅本地 `.gitignore`）。
- **`src/star_rail_pet/`**：真正的库代码；根目录或 `tools/` 只保留**几行入口**，避免再出现千行单文件。
- **拆分粒度**：`spine/` 与 `render/` 可分两阶段迁移（先 `mesh_scene + bones + atlas`，再 `gl_preview`）。

## 5. 后续功能挂载点（预留）

在 `run_gl_preview` 主循环中，顺序保持为：

1. `bones = apply_animation_to_bones(...)`  
2. `bones = apply_procedural_overlay(bones, ...)`（**新**）  
3. `worlds = compute_bone_worlds(...)`  
4. `apply_transform_constraints_translate_world(...)`  
5. 绘制  

程序层建议放在 `src/star_rail_pet/preview/procedural.py`（或 `pet/behavior.py`），**不**与 OpenGL 细节混在一起。

## 6. 迁移阶段（建议顺序）

1. **建目录**：`assets/argenti/`（或你的命名）+ `docs/架构规划.md`（本文件）+ 可选 `README.md`。  
2. **复制素材**：将 Spine `json + atlas + png` 放入 `assets/`，`.gitignore` 大 PNG 若需要。  
3. **建包 `src/star_rail_pet`**：先迁 `spine_anim_sample.py` → `anim/sample.py`，跑通 `python -m pytest` 或手动 `import`。  
4. **纵向切 `preview_spine_character.py`**：按上表拆到 `spine/*` 与 `render/gl_preview.py`，根入口改为 `tools/` 下薄脚本。  
5. **默认路径**：CLI `--dir` 默认改为 `assets/argenti`（相对仓库根）。  
6. **再实现**程序化交互（看向光标、点击等）。
