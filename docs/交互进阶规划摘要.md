# 交互进阶规划摘要

本文是 **`docs/交互与表情仲裁设计.md`** 的 **速览**：收 **信号分离、TTL/能量、与时间轮播的合并顺序**，便于实现时对照；**规则与参数表以仲裁设计正文为准**。

**互见**：**`docs/交互与表情仲裁设计.md`**（完整约定）；**`docs/agent协同开发对话纪要.md`** → **「动态与交互进阶」**、**「动态切换状态机」**（工程语境与踩坑）。

---

## 1. 统一出口与时间轮播

- **所有**改写 `track0` 的意图（时间轮播如 **`__cycleTick`**、流汗、星星眼、说话等）最终经 **同一仲裁模块**，由 **唯一路径** 调用 **`setAnimation(0, …)`**。
- **合并顺序**：每帧 **先跑仲裁器**（优先级、`talkUntil` 短路、冷却与残留锁）；**仅当仲裁允许「空闲/低优先级轮播」时**，再让时间轮播逻辑决定是否切入随机 emoji。细则见仲裁文档 **第 2～6、10 节**。

## 2. 信号分离（避免天然冲突）

| 能力 | 信号要点 |
|------|----------|
| 流汗 | **窗口**竖直速度 **`vy`**；**仅在判定为拖拽中时**采样（与光标在客户区内位移分离）。 |
| 星星眼 | 客户区光标晃动能量 + **`hitTestAtClientPoint`**；**拖拽进行中不累计**（冻结或强衰减能量）。 |
| 说话 | **`talkUntil` 内**对其它表情意图 **短路、不处理、不入队**。 |

## 3. TTL / 能量 / 残留锁（实现抓手）

- **流汗**：`sweatUntil` 延长、避免每帧重复 `setAnimation`；可选最短展示与结束冷却（仲裁文档 **第 4.1、5 节**）。
- **星星眼**：能量累积与阈值、触发后冷却与能量清空（**第 4.2 节**）。
- **残留锁**：流汗尾巴期屏蔽星星眼等；**意图不入队**（**第 5 节**）。

## 4. 建议实现顺序（与正文一致）

1. **动画仲裁器**：优先级 + `*_Until` / 冷却；所有 **`setAnimation(0, …)`** 经统一出口。  
2. **窗口 `vy` + 拖拽标志 → 流汗**（TTL 延长）。  
3. **`inside` + 晃动能量 + 冷却 → 星星眼**（拖拽中不累计）。  
4. **番茄钟 / `beginSpeech` 对齐为 talk 顶层**：`talkUntil` 内短路；结束清理输入侧累积。  
5. **时间轮播**：挂在仲裁器之后，仅在允许轮播时推进。
