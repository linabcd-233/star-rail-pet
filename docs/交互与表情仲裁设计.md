# 桌宠交互与表情仲裁设计

本文档整理「银枝桌宠」表情与输入交互的**产品规则**与**技术路线**，与 `pet-web` 侧实现约束对齐（**`main.ts`** 组装、**`spine/canvasApp.ts`** 单轨与轮播、**`pomodoro/runtime.ts`** 的 `beginSpeech` / `isSpeechLocked`、穿透与像素命中等）；供后续落地与仲裁收口时对照。

**互见**：**`docs/agent协同开发对话纪要.md`** → **「动态与交互进阶」**、**「动态切换状态机」**（时间轮播与仲裁合并顺序）；**`docs/交互进阶规划摘要.md`**（信号分离与实现顺序速览）。

---

## 1. 动作与动画映射（约定）

| 行为 | 动画名（约定） | 说明 |
|------|----------------|------|
| 默认呼吸 | `idel` | 常态兜底 |
| 过山车流汗 | `emoji_2` | 拖拽窗口竖直速度超阈值；持续满足则**延长**表现时长 |
| 星星眼 / 灿烂笑 | `emoji_1`（以实际素材为准） | 本体像素命中区内光标**晃动能量**达标；带**冷却** |
| 说话（番茄钟等） | `emoji_5` | 固定时长台词 + 气泡；**最高优先级、不可打断** |

---

## 2. 优先级（从高到低）

1. **番茄钟说话**（有意开启、频率低）— 独占，其它输入**不切换动画、不入队**。
2. **流汗**（拖拽高速 / 过山车）。
3. **星星眼**（本体内抚摸式晃动）。
4. **默认呼吸** `idel`。

仲裁原则：**单轨道 track0**，由统一模块根据优先级与 TTL 决定**何时** `setAnimation(0, name, true)`，避免同一表情被反复 `set` 导致闪烁或叠层。

---

## 3. 信号来源（避免天然冲突）

| 能力 | 推荐信号 | 说明 |
|------|----------|------|
| 流汗 | **窗口位置**（如 `outerPosition`）的竖直速度 `vy`；仅在**判定为正在拖拽**时采样 | 与「鼠标在客户区内位移」分离；左键拖窗时相对窗口的光标往往较稳，速度主要来自窗体移动 |
| 星星眼 | **窗口客户区坐标系**下光标位移 / **晃动能量**；且 **`hitTestAtClientPoint === true`** | 仅在「手型、命中本体」时累计 |
| 拖拽中手抖 | **拖拽进行中不累计星星眼**（或冻结/强衰减 `wiggleEnergy`） | 防止与过山车同时误触星星眼 |

---

## 4. 防抖 / 节流：用 TTL 与能量，不用重复触发

### 4.1 流汗（延长生存期）

- 维护结束时间戳 `sweatUntil`（或等价毫秒时间）。
- 每次判定「应流汗」：`sweatUntil = max(sweatUntil, now + SWEAT_EXTEND_MS)`。
- **仅当当前模式还不是流汗时**再切到 `emoji_2`，避免每帧 `setAnimation`。
- 可选：`SWEAT_MIN_ONCE_MS` 保证首次至少播一段；结束后可接短冷却 `SWEAT_COOLDOWN_MS`。

### 4.2 星星眼（能量 + 冷却）

- 在 `inside` 时：`wiggleEnergy += clamp(speed, 0, max) * dt`（或等价实现）；离开或静止时**衰减**。
- 超 `ENERGY_THRESHOLD` 且不在 `starCooldownUntil` → 触发 `emoji_1`，并进入冷却 `STAR_COOLDOWN_MS`。
- 触发后清空或衰减能量，避免连发。

### 4.3 番茄钟说话

- `talkUntil = now + durationMs`；`now < talkUntil` 内仲裁器对流汗/星星眼等**直接短路，不处理、不入队**。

---

## 5. 流汗「残留期」：锁，不排队

- **触发期**：高速拖拽持续则不断**延长** `sweatUntil`。
- **松手后的尾巴**：视为 **sweat 锁 / 残留 TTL** — 期间**屏蔽星星眼**等低优先级（是否屏蔽番茄由优先级表决定；当前约定番茄最高，仅当未在说话时讨论流汗/星星眼互斥）。
- **锁期间意图不入队**：避免解锁后连环触发；抚摸能量可衰减或冻结，解锁后不「补播」积压事件。
- 若希望「松手立刻能摸」：缩短锁时长、或提高星星眼阈值，而不是引入复杂队列。

---

## 6. 番茄钟说话：不可打断（已定稿）

- 进入：`triggerTalk` / `beginSpeech` → `emoji_5` + 气泡，`talkUntil` 内独占。
- 进行中：**忽略**流汗、星星眼、随机表情轮播等；**不排队**。
- **说话结束**：回到 `idel`（或约定默认态），再恢复其它判定；建议**清零或冻结** `wiggleEnergy`、拖拽速度缓存，避免刚结束瞬间误触。

---

## 7. 与 Tauri / 现有工程的兼容要点

- **点击穿透**：沿用 `setIgnoreCursorEvents` + 像素命中；交互逻辑不增加 `readPixels` 面积即可。
- **拖拽**：`startDragging()` 后窗体由系统移动，应用侧用**轮询窗口位置**（如 ~30Hz）估 `vy`；**仅在拖拽态**开启可降低 IPC/CPU。
- **现有代码**：番茄钟、气泡、`isSpeechLocked()`、`cycleMode` 等需逐步**收口到同一仲裁器**，保证「说话」与其它表情只从一个出口切动画。

---

## 8. 建议参数初值（需实测调手感）

| 项 | 建议量级 | 说明 |
|----|----------|------|
| `VY_THRESHOLD` | 约 900–1400 px/s（或 EMA 后阈值） | 过山车手感 |
| `SWEAT_EXTEND_MS` | 约 800–1200 ms | 持续满足则续命 |
| `SWEAT_MIN_ONCE_MS` | 约 1200 ms | 首次最短展示 |
| `SWEAT_COOLDOWN_MS` | 约 600 ms | 结束后的短冷却 |
| `ENERGY_THRESHOLD` | 约 160–260（与 dt 尺度相关） | 星星眼 |
| `STAR_COOLDOWN_MS` | 约 3000–6000 ms | 防连发 |
| 台词时长 | 与现有 `beginSpeech(durationMs)` 一致 | 固定展示 |

---

## 9. 建议实现顺序

1. **动画仲裁器**：优先级 + `*_Until` / 冷却；所有 `setAnimation(0, …)` 经统一出口。
2. **窗口 `vy` + 拖拽标志 → 流汗**（TTL 延长）。
3. **`inside` + 晃动能量 + 冷却 → 星星眼**（拖拽中不累计）。
4. **番茄钟 / `beginSpeech` 对齐为 talk 顶层**：`talkUntil` 内短路；结束清输入侧累积。

---

## 10. 状态转移（概念表，便于编码对照）

- **Talk**：仅允许到期 → `idel`；内部忽略 sweat / star / 轮播。
- **Sweat（含残留锁）**：不允许被 star 打断；不被 talk 打断的规则由优先级保证（talk 时根本不进 sweat 分支）。
- **Star**：不被 sweat 锁期触发；talk 时忽略。
- **Idle**：接受除 talk 外的合法升级（sweat / star）。

（若后续增加显式状态枚举，可在此表基础上扩展 `idle / sweat / sweat_tail / star / talk`。）
