# 银枝桌宠 — 技术文档

## 1. 可行性结论

| 方向 | 结论 |
|------|------|
| **桌宠形态** | 可行。Windows 上常用：无边框置顶窗口 + 透明/分层（PyQt6、Electron、或 pygame + `pygame.Window`/`SRCALPHA` 等方案）。 |
| **骨骼动画** | **数据已具备**：`1302.1a88ff13_ske.json` 为 **DragonBones 5.5** 骨架与动画列表；同目录 `1302.1a88ff13.json` 为 **Spine 4.2** 导出。二选一接入官方或社区运行时即可，不必从零解析曲线与网格。 |
| **贴图** | `1302.atlas` 引用 **`1302.png`**。若目录中缺少 PNG，线框预览仍可运行；**完整角色显示需补全图集 PNG**（与 atlas 同目录）。 |
| **交互（点击/拖拽/跟随）** | 可行，属于窗口级鼠标事件 + 状态机。 |
| **养成（随机词条）** | 纯逻辑，用 `random` 即可，与渲染解耦。 |
| **番茄钟 / 待办** | 可行；定时可用线程/`asyncio`/系统通知（如 `plyer`、Win10+ Toast）。 |

**风险点**：若坚持 **纯 Python + Pygame 自研 DragonBones 播放器**，工作量极大（网格变形、IK、曲线插值、换肤）。**推荐**：用 **Spine 官方运行时（C/C++）经绑定**、或 **DragonBones 的 JS/C++ 运行时嵌入 WebView/Electron**，或 **Unity/Godot 导出透明窗口**（工程量大但工具链成熟）。

## 2. 技术架构（建议分层）

```
┌─────────────────────────────────────┐
│  UI 壳层：菜单、番茄钟、待办、设置   │
├─────────────────────────────────────┤
│  桌宠核心：窗口、拖拽、置顶、多显示器 │
├─────────────────────────────────────┤
│  角色表现：骨骼动画状态机、音效      │
├─────────────────────────────────────┤
│  动画运行时：Spine / DragonBones     │
├─────────────────────────────────────┤
│  资源：ske/json、atlas、png、音频    │
└─────────────────────────────────────┘
```

- **表现数据**：待机 / 行走 / 交互 等映射到动画名（当前 ske 中可见如 `idel`、`emoji_*` 等，具体以工程内动画名为准）。
- **养成系统**：独立模块，通过事件通知 UI（例如“刷出一条风套词条”弹条），不阻塞动画线程。

## 4. 依赖与技术栈选型

### 4.1 最小验证（已实现）

| 依赖 | 用途 |
|------|------|
| **Python 3.10+** | 运行环境 |
| **pygame ≥ 2.5** | 窗口 + 骨骼**线框**绘制（不解析完整 DragonBones 动画曲线） |

安装：

```bash
pip install -r requirements.txt
```

### 4.2 正式桌宠（择一扩展）

| 方案 | 典型依赖/技术 | 说明 |
|------|----------------|------|
| **PyQt6 / PySide6** | `PyQt6`, `QTimer`, 可选 `QMediaPlayer` | 商业友好需注意许可；适合做半透明置顶窗与系统托盘。 |
| **Electron + Web** | Node, `pixi-spine` 或 DragonBones JS | 动画生态好，包体较大。 |
| **Godot 4** | GDScript / C# | 2D 骨骼工作流顺，导出桌面工程。 |
| **Spine 官方运行时** | C/C++ 或绑定语言 | 你已有一份 Spine JSON，可走 Spine 管线。 |

### 4.3 工具向功能

| 功能 | 可选依赖 |
|------|----------|
| 系统通知 | `plyer` 或 WinRT Toast |
| 配置持久化 | `pydantic` + JSON / `tomllib` |
| 打包 exe | `pyinstaller` / `cx_Freeze` |

## 5. 资源格式说明

- **`1302.1a88ff13_ske.json`**：`frameRate`、`armature`、`bone`、`slot`、`skin`、`animation` 等字段 → **DragonBones**。
- **`1302.1a88ff13.json`**：顶层含 `"spine":"4.2.40"` → **Spine** 骨骼 JSON（与 ske 二选一作为主数据源即可，避免混用两套逻辑）。
- **`1302.atlas` + `1302.png`**：Spine/DragonBones 共用风格的图集描述；**缺 PNG 则无法显示贴图**。

## 6. 预览脚本用法

在项目根目录：

```bash
# 骨骼线框（DragonBones ske；皮肤数据若损坏则只能看线框）
python preview_skeleton.py
python preview_skeleton.py "D:\path\to\1302.1a88ff13_ske.json"
python preview_skeleton.py --summary-only

# 带贴图的角色（使用同目录 Spine JSON + 1302.atlas + 1302.png，OpenGL 绘制 mesh）
pip install -r requirements.txt
python preview_spine_character.py --dir "D:\path\to\银技"

# 程序内循环播放骨骼动画（线性插值；不含 slot 附件切换 / IK / 网格变形 / 贝塞尔曲线）
python preview_spine_character.py --dir "D:\path\to\银技" --anim idel
python preview_spine_character.py --dir "D:\path\to\银技" --anim emoji_2

# 仅看整张图集缩略图（纯 pygame，无需 OpenGL）
python preview_spine_character.py --dir "D:\path\to\银技" --mode bitmap
```

说明：**DragonBones 的 `*_ske.json` 里皮肤常为占位，无法驱动贴图**；拼装预览依赖 **`1302.1a88ff13.json`（Spine 4.2）** 与 **`1302.png` 图集**。本仓库不按散图目录加载；修图或换皮应在 Spine 中重新打包 atlas 后再导出。

**若拼装失真、错层、色块、边缘糊：**  
- **UV**：JSON 里 mesh 的 `uvs` 是 Spine 的 *regionUVs*，必须结合 atlas 的 `offsets`（原始尺寸与裁剪偏移）与 `rotate` 换算；旧版脚本若直接用 `bounds` 映射会采到邻图，表现为错位与色块。当前 `preview_spine_character.py` 已按官方 `spMeshAttachment_updateRegion` 对齐。  
- **模糊**：`1302.atlas` 里常见 `scale:0.2`，表示图集按 **20% 分辨率** 导出，放大预览必然偏糊；需从工程 **导出更高分辨率** 图集（或去掉低倍率 scale）。  
- **混合**：若图集含 `pma:true`（预乘 Alpha），需用预乘混合；脚本会读 `pma:` 行。  
- **过滤**：默认与 atlas `filter:Linear` 一致；可加 `--nearest` 试最近邻（更硬、更像素，略减糊感但仍有锯齿）。

窗口内 **ESC** 或关闭窗口退出。

## 7. 后续里程碑（建议顺序）

1. 将 `1302.png` 与 atlas、ske 放入仓库 `assets/argenti/`，确认图集路径一致。  
2. 选定 **Spine 或 DragonBones** 其中一条运行时，跑通**带贴图**的默认姿态。  
3. 接入 **idel** 等动画播放与状态切换。  
4. 叠加无边框置顶窗与拖拽。  
5. 番茄钟、待办、随机词条作为独立模块挂载。
